Splash Page

Technical Description

☐ HTML Structure
"div class="background-image"" This is the container for the entire splash screen. It holds the background image and ensures it covers the entire viewport using background-size: cover.
"div id="main""" Acts as a wrapper for the animated elements (#box1 and #box2). It ensures proper positioning and overflow handling.
"div id="box1""" Represents the animated block that moves from the bottom to the top and then to the right. It uses a background image that changes during the animation.
"div id="box2""" Contains the text content (title, slogan, description, and student details). It is initially hidden and slides in from the left with a fade-in effect.
☐ CSS Styling and Animations
background-image: Used in .background-image to set the background image for the splash screen. The background-size: cover ensures the image scales to fit the screen.
position: absolute: Used for #box1 and #box2 to position them precisely within the #main container.
animation: Key CSS property used to create animations:
slideUp: Moves #box1 from the bottom of the screen to the top.
moveRight: Moves #box1 to the right and changes its background image.
slideIn: Slides #box2 (text content) from the left into view.
fadeIn: Gradually makes #box2 visible by increasing its opacity.
@keyframes: Defines the animation sequences for slideUp, moveRight, slideIn, and fadeIn.
font-family and font-size: Used to style the text content (#box2) with custom fonts and sizes for better readability and visual appeal.
opacity: Used in #box2 to initially hide the text and then reveal it during the fadeIn animation.
background-color: rgba(255, 255, 255, 0.2): Adds a semi-transparent background to the student details (#box2 .text4) for better contrast against the background image.
☐ Purpose of Key Tags and Styling
"div" Used to create containers for different sections of the splash screen, ensuring a modular and organized structure.
position: absolute: Allows precise positioning of elements relative to their parent container.
animation and @keyframes: Create smooth and engaging animations to guide the user's attention and make the splash screen visually appealing.
background-image: Sets the visual theme of the splash screen and provides context for the content.
font-family and font-size: Ensure the text is readable and aligns with the design aesthetic.
opacity and transition: Used to create fade-in effects, making the text appear smoothly.

Accessibility

☐ Semantic HTML While the page primarily uses div tags for structure, the content inside #box2 is organized into logical sections (title, slogan, description, and student details). This helps screen readers interpret the content correctly. Contrast and Readability
The text color (color: whitesmoke) contrasts well with the dark background image, ensuring readability for users with visual impairments
The semi-transparent background (background-color: rgba(255, 255, 255, 0.2)) for the student details improves contrast without overwhelming the background image.
☐Animation Considerations
The animations are smooth and not overly fast, reducing the risk of triggering motion sensitivity in users with vestibular disorders.
The animations are purely visual and do not interfere with the functionality or accessibility of the content.
Responsive Design
The splash screen uses height: 100vh and width: 100% to ensure it scales properly across different screen sizes and devices.
The text width is limited to 50% (width: 50% in #box2) to prevent it from becoming too wide on larger screens, improving readability.
Alternative Text for Images
While the background images (#box1 and .background-image) are decorative and do not require alternative text, the use of background-image instead of img tags is appropriate here since the images are not content-critical.

Volunteer Page

Technical Description

☐ HTML Structure div class="bgfixed": Creates a fixed background gradient that covers the entire page, providing a consistent visual theme. div class="heading": Contains the main heading and a dropdown for sorting/filtering volunteer programs. div class="container": Wraps all the volunteer program cards (div class="card1"). It uses flexbox to arrange the cards in a responsive grid. div class="card1">: Represents individual volunteer program cards. Each card contains an image, a title, and hidden content that appears on hover. div class="ratings-section": Displays the overall rating and a link to view all reviews. div class="reviews-section": Contains user reviews displayed in a grid layout. div class="feedback-section": Provides a form for users to submit feedback. div class="rating-input": Allows users to rate their experience using emojis.
☐ CSS Styling and Animations
background-image: linear-gradient: Used in .bgfixed .bg to create a gradient background that spans the entire page. flexbox: Used in .container to arrange the volunteer program cards in a responsive grid. position: absolute: Used in .card1 .content and .card1 .title to position elements precisely within the card. hover effects: Applied to .card1 to reveal hidden content when the user hovers over the card. transition: Used for smooth animations on hover (e.g., enlarging images, revealing content). box-shadow: Adds depth to cards and sections, making them visually appealing. border-radius: Rounds the corners of cards and sections for a modern look. textarea and buttons: Styled in the feedback section to match the website's theme and provide interactive hover and click effects.
☐ Purpose of Key Tags and Styling
div: Used to create containers for different sections, ensuring a modular and organized structure. flexbox: Ensures the volunteer program cards are responsive and adapt to different screen sizes. hover effects: Enhances user interaction by revealing additional information when hovering over cards. box-shadow and border-radius: Improve the visual appeal of cards and sections. textarea and buttons: Provide a user-friendly way to submit feedback and rate experiences.

Accessibility

☐Semantic HTML The page uses div tags extensively, but the content is logically organized into sections (e.g., heading, container, ratings, reviews, feedback). This helps screen readers interpret the content correctly. ☐ Contrast and Readability The text color (color: azure) contrasts well with the dark gradient background, ensuring readability for users with visual impairments. The semi-transparent background for the feedback form (background-color: #fffbd5) improves contrast without overwhelming the background. ☐ Interactive Elements Buttons and links have hover and active states, providing visual feedback for users interacting with them. The feedback form includes a textarea with a placeholder for instructions, making it clear what users should do.

Challenges and Lessons Learned

The project highlighted the importance of integrating accessibility and performance considerations early in the design process. Using semantic HTML and ARIA attributes improved accessibility, while tools like Lighthouse and WAVE helped identify compliance gaps. Testing on multiple devices and browsers ensured a consistent user experience. Prioritizing responsive design and optimizing images and code significantly enhanced performance. These lessons underscored the value of proactive planning and continuous testing to meet compliance standards effectively.

Compliance

Ensuring compliance with modern web standards and accessibility guidelines was a significant challenge. Balancing design aesthetics with accessibility requirements, such as proper contrast ratios and keyboard navigation, required careful planning. Additionally, optimizing performance for fast load times while maintaining high-quality visuals and animations was tricky. Cross-browser compatibility also posed challenges, as some CSS properties and animations behaved differently across browsers.
